# 谷歌 2.0：从搜索引擎到平台帝国
### Acquired 播客 Google 第二集 — 中文译稿

---

## 第一章：纯粹玩法的终结

虚构角色罗斯·汉内曼（Russ Hanneman）——来自 HBO 剧集 Silicon Valley——曾留下这样一段话，在科技圈流传至今：

> "如果你展示了营收，人们就会问有多少？而数字永远不会令他们满足。那个曾经百倍、千倍增长的公司，转瞬之间就沦为只有两倍增长的狗。但如果你没有营收，你就可以说自己是 pre-revenue，一个潜力无限的纯粹玩法（pure play）。关键不在于你赚了多少，而在于你值多少。最值钱的是什么？亏损的公司。"

对谷歌而言，营收从来不是问题。但"纯粹玩法"这顶帽子，在 2004 年到 2006 年间却成了一把双刃剑。

2004 年秋，谷歌上市，股价在两个月内近乎翻倍。华尔街为之疯狂：AdWords 搜索广告商业模式，让每个人都相信谷歌已经破解了互联网货币化的密码——网络用户越多，搜索越频繁，广告越密集，钱也就越多。逻辑清晰，简单得令人喜爱，这就是纯粹玩法。

然而，2006 年 1 月，谷歌发布 2005 年全年财报，局面骤然逆转。

全年营收 $6.1 billion，较 2004 年的 $3.1 billion 几乎翻倍——按理说，华尔街应该欢呼雀跃。但利润持平，盈利能力下滑。谷歌正在将资金砸向 Gmail、Maps，以及即将上线的 Google Docs；那一年晚些时候，它还以 $1.6 billion 收购了 YouTube。

消息一出，股价单日暴跌 27%。

媒体人史蒂文·列维（Steven Levy）在 *In the Plex* 中写道：外界对谷歌在搜索之外一切尝试的看法，只有一种——这家公司像一个醉醺醺的杂耍演员，把球往空中乱扔。投资者想要的是纯粹玩法，而谷歌却在往里掺杂质。

谷歌为什么要这样做？

答案，藏在每一个产品背后的故事里。

谷歌联合创始人拉里·佩奇（Larry Page）的逻辑一直简单直接：如果我们能构建出更好的产品，而这些产品存在于网络之上，那就对谷歌有利，我们就应该去做。

但这层产品哲学之下，还有一条更紧迫的战略逻辑——一场谷歌与微软（Microsoft）之间无声的存亡博弈。

彼时，谷歌超过 90% 的搜索请求来自 Windows 电脑，其中 90% 又是通过微软的 Internet Explorer 浏览器发起的。谷歌掌握了互联网最有价值的入口——搜索，但这个入口架在微软的平台之上。换句话说，谷歌整个商业帝国的根基，建立在微软的容忍之上。

最清醒看到这一危险的人，是谷歌 CEO 埃里克·施密特（Eric Schmidt）。在加入谷歌之前，他曾在 Novell 担任 CEO，而 Novell，正是被微软亲手摧毁的那家公司。

谷歌的反制策略由此清晰：如果互联网用户能够爱上那些只需要浏览器就能运行的网络应用，微软对 Windows 和 IE 的掌控就会逐渐失去意义。而构建这套防御体系的方式，是一个接一个地打造出令用户欲罢不能的产品。

第一个，叫做 Gmail。

## 第二章：Gmail——改写规则的一封邮件

**愚人节的真实革命**

2004 年 4 月 1 日，愚人节。

谷歌发布公告：推出全新电子邮件服务，完全基于浏览器运行，支持任何设备随时访问，内置谷歌搜索，无需手动整理文件夹，提供 1 吉字节（1GB）的免费存储空间，用户几乎永远不必删除任何邮件——一切全部免费。

听上去像是一个笑话。彼时，Hotmail（微软旗下）仅提供 2MB 的免费存储，Yahoo Mail 提供 4MB。1GB，是这个数字的 250 到 500 倍。

当时的市场格局是：微软向企业销售高价邮件服务；Hotmail、Yahoo Mail 和 AOL 则向个人用户提供免费网页邮箱，但存储极为有限，用户不得不像整理实体信件一样反复归档、删除。

史蒂文·列维（Steven Levy）在 *In the Plex* 中记录了一个细节：Gmail 推出后不久，一位记者在纽约采访比尔·盖茨（Bill Gates）时提及此事。盖茨勃然大怒——他认为给用户提供如此大量的存储空间根本就是浪费，是"道德上令人反感的"事情。在他看来，邮件就应该好好管理、及时清理；而谷歌的做法，是在刻意做错误的事。

直到 Gmail 出现，人们才意识到：电子邮件从来不需要被管理，存储成本的下降早已注定了一种新的使用范式。

**保罗·布赫海特（Paul Buchheit）与 AJAX 的诞生**

这个故事要从 1996 年说起。

保罗·布赫海特（Paul Buchheit）当时是 Case Western Reserve University 的在读学生。这所位于克利夫兰的大学，是美国最早在宿舍和校园全面部署宽带网络的高校之一。在宽带已经无处不在的校园环境中，保罗提前活在了 15 年后的未来——他意识到，如果电子邮件必须下载到本机才能阅读，本质上就是个累赘。信息应该随时在指尖可及，比特的传输成本正在趋近于零，为什么邮件不能直接存在于网络上？

他在大学期间便构建了一个网页版邮件的原型。

2001 年，谷歌上市前，拉里·佩奇（Larry Page）感到公司运转得太慢，做出了一个震惊全员的决定：裁撤所有工程部门的管理层，直接与每一位工程师面谈，鼓励他们独立全栈负责某个产品。拉里和工程负责人韦恩·罗辛（Wayne Rosing）找到保罗时，知道他此前对网页邮件有强烈的执念，于是向他提出了一个模糊的构想。就这样，Gmail 有了起点。

保罗后来透露，开发 Gmail 的动机之一，是让谷歌变得更具"粘性"——即便未来出现了下一个谷歌，用户也有理由继续与这家公司保持关系。

Gmail 代码的第一粒种子，意外来自谷歌的第一笔收购：Deja News——一个包含所有旧 Usenet 帖子的语料库，后来成为 Google Groups。保罗参与了对这些帖子的实时索引工作，随后他将同样的搜索逻辑应用到了自己的个人邮箱。Gmail 最早的形态，就是一个搜索他个人 Unix 邮件目录的搜索框。

接下来，保罗开始研究如何在浏览器中构建一个真正流畅的邮件界面。彼时，Hotmail 和 Yahoo Mail 每次操作都会重新加载整个页面，体验极差。在探索 JavaScript 的过程中，他发现了一个鲜为人知的特性——XMLHttpRequest。这个 API 允许网页在不重新加载整个页面的情况下，自动从服务器获取新数据。

保罗立刻意识到：这是黄金。

这是 AJAX（Asynchronous JavaScript and XML，异步 JavaScript 与 XML）的诞生时刻。

颇具讽刺意味的是，XMLHttpRequest 最初由微软实现，用于 Outlook 的网页版 Outlook Web Access——这个为微软邮件客户端设计的 API，最终成为了颠覆微软的武器。

当人们意识到，借助这一技术，可以在浏览器中构建出与本地安装程序同样强大的应用——曾经需要去零售商店购买安装盘的软件，如今在浏览器里就能运行——这堪称与谷歌搜索同等量级的技术顿悟。Gmail 是有史以来第一个被全球广泛采用的 AJAX 应用，它重新定义了"网站"与"网络应用"的边界，也标志着 Web 2.0 时代正式开幕。

Gmail 开发历时三年，从 2001 年持续到 2004 年。一切都是全新领域：没有成熟的 JavaScript 框架，没有现成的 AJAX 开发工具，一切从零探索。

**邀请机制与病毒式传播**

Gmail 公开发布时，谷歌仅向外部发出了 1000 份邀请，定向发给具有影响力的人士和记者。每位用户获得数量有限的邀请名额，可以分享给朋友。

这一安排看似低调，实则精妙：它制造了稀缺感，营造出"进入知情者专属圈子"的氛围；病毒式口碑传播被自动激活；邀请名额甚至在 eBay 上被炒至平均 $150 一个。这份礼物，需要产品质量来兑现——而 Gmail 做到了。这项服务真实改变了人们每天查看邮件的方式，通过了拉里·佩奇所说的"牙刷测试"：一个每天使用一两次、已成为生活习惯的产品。

**内容匹配广告：AdSense 的种子**

如何从免费的 Gmail 中赚钱？保罗在开发阶段进行了一个实验：连接到谷歌的广告数据库——专为搜索结果页面准备的广告库——将广告与邮件内容做关键词匹配，并将匹配到的广告展示在邮件页面旁边。

效果出人意料地好：这些搜索广告，在收件箱语境下同样相关。内部反响却是轩然大波——谷歌员工得知自己的邮件被"扫描"后勃然大怒，"你们在监视我们的邮件？"拉里和谢尔盖（Sergey Brin）却对此极为兴奋。

更值得一提的是：这个实验发生在 AdSense 诞生之前。内容匹配广告先在 Gmail 内部得到验证，随后才以 AdSense 的形态推向了外部网站发布商。Gmail 为谷歌最重要的广告产品之一，埋下了最初的种子。

**战略意义：对抗微软的第一道护城河**

Gmail 的真正价值，远不止于产品本身。

谷歌整个商业大厦，建立在微软容忍的前提下。一旦微软决定将 IE 的默认搜索换成自家产品，谷歌将无力反制。对抗这一风险的唯一方式，是让普通用户对网络应用产生深度依赖——一旦他们爱上了只在浏览器里运行的服务，微软对 Windows 的掌控就会失去筹码。Gmail 是这个计划的第一块基石。

从最初 1000 位种子用户出发，Gmail 在此后 20 年间扩张至超过 20 亿活跃用户，至今仍是全球最佳的电子邮件服务。它的成功也令拉里、谢尔盖和施密特看到了一种可复制的模式：我们还可以把什么东西放进浏览器？

这个问题，接下来的答案，叫做 Google Maps。

## 第三章：地图与文档——Web 2.0 的两根支柱

**Google Maps：三家公司拼出的地图帝国**

2003 年，Gmail 尚未发布，谷歌一位年轻的副产品经理（APM）——布雷特·泰勒（Bret Taylor）——找到了拉里·佩奇。他说：AOL 刚刚以 $1 billion 收购了 MapQuest，而据他所知，Yahoo 即将大举进入地图市场。拉里的反应直截了当："好啊，这是网络产品吗？当然是。去做吧。"

布雷特·泰勒此后的轨迹同样耀眼：FriendFeed 创始人、Facebook CTO、Salesforce 联席 CEO、OpenAI 董事会主席。但这一切，都始于他在谷歌推动了 Google Maps 的诞生。

彼时的地图市场，MapQuest 和 Yahoo Maps 都是静态网页，核心功能只有一个——生成供用户打印的行车路线，打印纸上附带广告，本质是披着数字外衣的报纸广告。

谷歌的目标截然不同。谷歌收购了一家澳大利亚公司 Where2 Technologies，创始人是两兄弟——拉尔斯·拉斯穆森（Lars Rasmussen）和延斯·拉斯穆森（Jens Rasmussen）。他们已经构建了一款实时交互式地图应用，但当时是安装在桌面上的。拉里提了一个要求：把它搬到网络上。两兄弟用大约三周时间，重写并重新架构了整个应用，几乎独立发现并实现了谷歌内部正在为 Gmail 研发的大量 JavaScript 和 AJAX 特性。

谷歌同期还收购了两家公司：ZipDash（实时路况数据）和 Keyhole（后来成为 Google Earth）。

2005 年 2 月，Google Maps 正式上线，引发轰动。上线首日还有一个著名的彩蛋：打开 maps.google.com，用户能看到浩瀚大西洋中漂浮着一个孤独的北美大陆——欧洲、亚洲、非洲，全部付之阙如。这是团队对"最小可行产品"的诠释：先发布能用的，剩下的以后再说。

**Maps API 点燃 Web 2.0**

2006 年，谷歌发布 Google Maps API，这才是真正引爆 Web 2.0 时代的导火线。

开发者发现，可以轻易地将 Google Maps 嵌入自己的产品，创造出所谓的"mashup"——混搭应用。Zillow（房产平台）、Uber、DoorDash、Airbnb，这些今天已是行业巨头的公司，都直接建立在 Google Maps API 之上。没有它，这些公司根本不可能存在。

长期以来，Maps API 几乎免费向开发者开放，上限极高——这是谷歌"先做产品，后想商业"的典型逻辑。谷歌是唯一一家愿意花上数百亿美元，从零开始绘制整个地球地图的公司：派车队拍摄街景，建立众包更新体系，解决实时数据获取难题。这是一个极其艰巨的数据与工程问题，但谷歌义无反顾地扛了下来。

如今，Google Maps 拥有超过 20 亿活跃用户，估算年营收超过 $5 billion，甚至可能高达 $10 billion。

**Google Docs 与 Spreadsheets：只有谷歌才能做到的战略棋局**

谷歌随后推出了在线文档和电子表格工具，直接冲击微软 Office。

过去，所有试图与微软竞争生产力软件的公司都以失败告终：WordPerfect、Lotus Notes、Lotus 1-2-3，无一例外。但谷歌的切入点截然不同——它并非要做一个更好的 Word 或 Excel，而是利用 AJAX 技术实现了一件此前从未有人做到的事情：**实时多人协作**。

Google Docs 的前身是一家创业公司 Writely，由山姆·席拉切（Sam Schillace）和两位联合创始人创办。他们相信，只要把网络应用的交互性推到极致，就能在浏览器里实现真正的实时文档协作——这在历史上还从未有人做到过。Google Spreadsheets 的前身公司由乔纳森·罗谢尔（Jonathan Rochelle）创立，他对此表达得更为直接：当 Sheets 团队起步时，能否在浏览器中让两个人同时编辑同一张表格，仍是一个悬而未决的问题。

谷歌分别收购了这两家公司，Writely 于 2006 年 3 月完成收购。

为什么只有谷歌才能做这件事？原因是多重的。

**基础设施成本**：对谷歌来说，运行 Docs 和 Sheets 的服务器增量负荷相比 Search 几乎微不足道。它们已经拥有了支撑数十亿搜索请求的基础设施，多开一项文档服务的边际成本，几乎可以忽略不计。

**无需盈利**：谷歌不需要从 Docs 或 Sheets 赚钱。微软几乎锁定了所有企业的 Office 预算，通过企业协议收取高额费用，这是微软商业模式的核心。谷歌对此毫不在意——它只需要用户留在网络上继续使用网络产品，然后搜索，然后广告。

**战略干扰**：免费提供 Docs 和 Sheets，迫使微软不得不回应——"你们的 Office 网络版在哪里？为什么 Word 在线版还不支持协作？"这让微软陷入两难：把 Office 搬到网络上意味着吞噬自己的软件销售，而不搬又意味着被市场抛弃。谷歌用极低的成本，制造了一个对微软来说代价极高的战略困境。

时至今日，这场博弈的结果颇为微妙：Google Workspace 的用户数远超 Microsoft Office（各产品月活跃用户均在数亿至数十亿级别），但微软的生产力与商业流程业务年营收高达 $120 billion，谷歌的云业务（含 Workspace）整体不过 $50 billion 出头。谷歌拥有用户，微软拥有钱——而谷歌对此感到完全满意。

这个故事还有一个令人莞尔的尾声：Writely 创始人山姆·席拉切后来在谷歌管理过 Docs、Sheets 和 Maps 等多个产品，如今他是微软的副 CTO。

## 第四章：YouTube——史上最佳收购的漫长证明

**错误的开始：Google Video**

2003 年，就在 Gmail 还未发布之际，谷歌便开始了视频领域的探索，项目名为 Google Video。它起源于广告部门，动机很明确：电视广告是当时全球最大的广告市场，而视频信息显然符合"组织世界信息"的使命。

然而，Google Video 从一开始便方向错了。

它把重心放在传统电视内容上，通过闭路字幕数据建立搜索索引，帮助用户找到电视节目——但无法直接播放，只是一个指向外部内容的搜索入口，连播放器都没有。与此同时，它对用户上传的普通视频毫无兴趣，在版权问题上极为谨慎，每一段内容都需要人工审核，往往要等一两天才能上线。

这样的产品，注定无法应对即将到来的用户生成内容（UGC）时代。

**三个 PayPal 前员工改变了一切**

2005 年初，三位来自 PayPal 黑手党的年轻人——查德·赫利（Chad Hurley）、贾韦德·卡里姆（Jawed Karim）和史蒂夫·陈（Steve Chen）——离开 PayPal，创建了 YouTube。

这个起点颇为戏剧性：YouTube 最初的定位是视频相亲网站，标语是"Tune in, Hook up"，他们甚至在旧金山湾区的 Craigslist 上投放广告，邀请"有吸引力的女性"录制视频上传——结果应者寥寥。幸运的是，他们随即转向，将 YouTube 变成了一个任何人都可以上传任何内容的通用视频平台。

还有一个鲜为人知的细节：查德的岳父是吉姆·克拉克（Jim Clark）——Silicon Graphics（SGI）和 Netscape（网景）的创始人。作为硅谷生态的最佳领航者，克拉克帮助这支年轻团队穿越了投资人和媒体的迷宫。

YouTube 的天才之处在于三点：

**极低的上传门槛**。任何人，任何时间，任何内容，上传即发布，服务器处理完成后立刻上线，不进行版权人工审查——这是 Google Video 永远无法做到的。

**极低的观看门槛**。内置流畅的网页播放器，用户打开链接即可收看，没有任何额外步骤。

**嵌入式传播机制**。YouTube 视频可以被嵌入任何网站，用户在别处看到视频，点击即播，传播成本趋近于零。

此外，YouTube 从早期便拥有强大的搜索功能，几乎从一开始就成为了世界第二大搜索引擎，仅次于谷歌本身。

2005 年，Saturday Night Live 的一段喜剧短片《Lazy Sunday》被用户上传至 YouTube，这一个视频让 YouTube 的流量在短期内暴增 83%。

YouTube 很快从 Sequoia（红杉资本）那里获得了融资，领投人鲁洛夫·博萨（Roelof Botha）是 PayPal 出身的老朋友。但快速增长随即带来了快速燃烧的现金危机。

**版权危机与被迫出售**

视频存储、带宽传输、编码处理——这三项成本随着用户规模线性增长，且没有任何收敛的迹象。更致命的是版权问题：用户大量上传受版权保护的内容，包括 NBC、CBS、Viacom 等媒体巨头的节目，诉讼威胁接踵而至，其中最著名的是来自 Viacom 的重磅诉讼。

此时的 YouTube 走到了一个关键路口：产品大获成功，但作为一家资源有限的独立创业公司，它无法同时应对版权博弈、基础设施扩张和内容谈判。2006 年 11 月——距 YouTube 产品上线仅 18 个月——谷歌以 $1.65 billion 的谷歌股票收购了这家公司。

这笔交易并非没有争议。谷歌 CFO 帕特里克·皮谢特（Patrick Pichette）事后表示"永不再犯"，称这是公司最后一次用股票作为收购对价。毕竟，谷歌市值在交易完成后的数年间上涨了 20 倍——如果当初用现金支付，省下的收益将是一个天文数字。

收购完成后，YouTube 的状况一度令谷歌内部颇为头疼。当时年营收约 $30 million，年亏损约 $1 billion——换算下来，大约每一次视频播放，谷歌就要亏损一分钱。时任 CFO 甚至私下探讨过能否将 YouTube 转卖给出价方。YouTube 在谷歌内部被广泛称为"第一个错误"。

**从亏损到 $50 billion：惊天逆转**

然而，历史最终给出了截然不同的答案。

2009 年，YouTube 广告营收实现三倍增长，公司开始扭亏为盈。2012 年，外部估算营收约 $4 billion，基本达到收支平衡。此后的关键转变，发生在 2013 年到 2015 年间：

YouTube 将内部北极星指标从"播放次数"切换为"观看时长"，目标确立为"让用户打开 YouTube，就能沉浸地看上 15 分钟"。移动端的兴起带来了大量低意图用户——他们打开 App 不是为了寻找某个特定视频，而是交给算法去推荐——算法由此获得了更大的发挥空间。与此同时，YouTube 成为互联网上最早向内容创作者分享广告收入的平台，大约将 50% 的广告收入分给创作者。这一安排让平台盈利周期拉长了将近十年，但也构建起了一个无可替代的创作者经济生态。

到 2024 年，YouTube 广告营收达到 $36 billion，加上 YouTube Premium、YouTube Music、NFL Sunday Ticket 等订阅收入，总营收突破 $50 billion，超越 Netflix（$39 billion），成为仅次于迪士尼的全球第二大媒体公司。研究机构 Moffett Nathanson 估算，若 YouTube 独立上市，市值约 $500 billion；即便以传统媒体估值方法折算，保守估计也超过 $200 billion。

这是一笔以 $1.65 billion 换来 $500 billion 潜在价值的交易。Acquired 将最初的评级从 C 提升至 A+，并称之为"史上最伟大的收购之一"。

还有一个颇具讽刺意味的结语：谷歌在社交媒体领域全面失败，但整个社交媒体行业却最终朝着 YouTube 的方向演化——无论是 Instagram Reels 还是 TikTok，用户看到的都是来自陌生人的短视频。谷歌什么都没有放弃，只是时间验证了它做对了什么。

## 第五章：DoubleClick 与广告帝国的版图扩张

**DoubleClick：一家在互联网泡沫中死而复生的公司**

2007 年 4 月，仅仅收购 YouTube 五个月后，谷歌再次出手，以 $3.1 billion 现金收购了展示广告巨头 DoubleClick——这一次，特别说明不再用股票支付。

DoubleClick 的历史可以追溯到 1995 年，由凯文·奥康纳（Kevin O'Connor）和德怀特·梅里曼（Dwight Merriman）在纽约创立，早于谷歌诞生。它最初的使命是帮助广告主跨网站投放广告（广告服务器 ad server），同时建立广告网络（ad network）。1998 年成功上市，成为互联网泡沫时代的明星公司。

然而，.com 泡沫破裂后，DoubleClick 的客户中 70% 不仅流失，而且直接倒闭——这些客户大多是风险投资扶持的互联网创业公司。品牌广告的数字化迁移尚未发生，整个展示广告市场几乎陷入停滞。DoubleClick 将广告网络部门以不到 $15 million 的价格出售，只剩下广告服务器软件苟延残喘。

2005 年，私募股权公司 Hellman & Friedman 和 JMI Management 以约 $1 billion 收购了这家公司。随后，新任 CEO 大卫·罗森布拉特（David Rosenblatt）上任，带来了一位名叫尼尔·莫汉（Neal Mohan）的产品与战略负责人——这个名字，如今是 YouTube 的 CEO。

私募股权运营期间，DoubleClick 推出了一个革命性的新产品：**广告交易所（ad exchange）**。这个概念彻底改变了数字广告的底层架构：它允许来自不同广告网络的需求方在同一市场上实时竞价，买卖展示广告库存。更进一步，它可以让广告主在实时基础上，与出版商的直接销售广告进行竞价，颠覆了此前各自为政的广告买卖格局。这实质上创造了现代程序化展示广告（programmatic display advertising）的基础设施。

**谷歌为何必须得到 DoubleClick**

理解这笔收购，需要理解谷歌彼时的处境。

谷歌已经是搜索广告的绝对霸主，但在展示广告领域却处于相对弱势：AdSense 面向中小发布商，自助服务，缺乏与大型广告代理商的深度绑定；谷歌在哲学上抵制第三方 Cookie，无法实现频次控制等高级功能；谷歌与顶级广告代理商之间缺乏直接的资金流对接，错失了流向大品牌的巨额预算。

DoubleClick 恰好拥有谷歌缺失的一切：与麦迪逊大道广告代理商深度整合的资金管道，行业第一的广告服务器软件，以及全新的广告交易所基础设施。

然而，这笔收购的关键动因，并非业务协同，而是竞争阻断。

谷歌内部销售负责人蒂姆·阿姆斯特朗（Tim Armstrong）偶然在西雅图撞见了 DoubleClick 的高管团队——这家总部位于纽约的公司，不远千里出现在西雅图，只有一个可能的理由：微软正在谈判收购。消息一经确认，谷歌立刻进入应急模式。

谈判演变成了一场四方争夺战：Yahoo、微软、AOL 和谷歌同时在场竞价。谷歌提交了 $3.1 billion 的意向书，并附上一个至关重要的条款——"风雨无阻条款"（hell or high water clause）：谷歌承诺无条件完成交割，不进行实质性尽职调查，不附加任何其他条件。这是明确的信号：现金在手，立即成交。DoubleClick 当即签字。

交易完成后，微软随即以 $6 billion 购入了展示广告市场的第二名 AQuantive——是 DoubleClick 成交价的两倍，换来的却是更差的资产。谷歌已然锁定胜局。

**搜索业务的进化：成为全球最大广告商**

与此同时，谷歌的核心搜索业务也在持续进化。

2003 年至 2008 年间，谷歌将搜索索引的更新频率从三个月一次逐步压缩到接近实时，推出了 Google Images、Google News、Google Books、Google Scholar、Google Suggest（搜索自动补全）和 Universal Search（综合搜索）。2005 年，谷歌开始将登录用户的历史搜索记录纳入结果个性化。2012 年，知识图谱（Knowledge Graph）上线，在搜索结果右侧提供实体信息摘要。

营收数字也呈现出令人叹为观止的增长曲线：
- 2004 年：$3 billion
- 2005 年：$6 billion（翻倍）
- 2007 年：$16.5 billion

2007 年，谷歌成为全球最大的广告销售商——不仅是数字广告，而是超越了电视、报纸、户外在内的所有媒介形式。此后每一年，谷歌都保持着这一头衔，直至今日。

## 第六章：Chrome——拯救开放网络的浏览器

**潜伏七年的计划**

Chrome 的故事，要从 2001 年说起。

那一年，拉里·佩奇和谢尔盖·布林便想建立一款属于谷歌的浏览器。理由显而易见：谷歌的一切都依赖于 Internet Explorer，而 IE 掌握在微软手中，这是一个随时可能被引爆的战略地雷。但当时的 CEO 埃里克·施密特坚决反对。用他的原话来说："2001 年，我不想去惹那头巨兽。"

于是，谷歌选择了另一条路：成为 Mozilla 基金会的主要资助者。Mozilla 是从 Netscape（网景）分拆出来的非营利组织，正在打造开源浏览器 Firefox。谷歌支付流量获取费用（TAC），让 Firefox 将谷歌搜索设为默认引擎，实质上是通过资助 Firefox 来对抗 IE。

随后，谷歌开始悄悄从 Mozilla 挖人，理由是：这些工程师去谷歌后将继续做同样的事情，只是薪酬更高，还能获得谷歌股票期权。这批来自 Mozilla 的工程师，在谷歌内部形成了一个"客户端产品组"。2004 年，谷歌从麦肯锡挖来了一位名叫桑达尔·皮查伊（Sundar Pichai）的人来领导这个团队。

正如施密特后来所说，这批工程师在谷歌内部静静地存在了好几年，"就像一个休眠的特工细胞，随时准备在微软威胁真正来临时被激活"。

这一天，终于在 2008 年到来。

**微软觉醒，谷歌反击**

2008 年 2 月，微软宣布以 $44 billion 收购 Yahoo，试图用合并后的力量正面挑战谷歌搜索业务。这只靴子终于落地。

幸运的是，Yahoo CEO 杨致远（Jerry Yang）拒绝了这个报价——在公司史上最糟糕的企业决策之一中，他把 $44 billion 拱手让给了时间。两年后，微软以 $1 billion 的协议取得了 Yahoo 搜索的授权，而 Yahoo 最终以单位数十亿美元的价格卖给了 Verizon。

但谷歌没有在这场运气里坐以待毙。它的反击，早已蓄势已久。

2006 年，谷歌正式启动 Chrome 研发。2008 年 9 月初——雷曼兄弟倒塌的前一周——Chrome 正式面向公众发布。谷歌选择了一种极为"谷歌式"的发布方式：委托著名漫画家斯科特·麦克劳德（Scott McCloud）创作了一本数字漫画册，作为 Chrome 的技术说明书，专门面向能够理解"进程独立"和"沙盒安全"的技术爱好者。

**Chrome 的四项技术突破**

**V8 JavaScript 引擎**：Chrome 内置了一个全新的超高性能 JavaScript 虚拟机——V8。它将 JavaScript 的执行速度提升了数个数量级，让大型 Web 应用在浏览器里能够流畅运行。这对于谷歌这家靠 AJAX 构建了全部产品体系的公司而言，是最核心的战略需求。

**标签进程隔离**：在 Chrome 之前，如果某个网页 Tab 崩溃，整个浏览器会随之关闭，所有打开的页面全部丢失。Chrome 将每个 Tab 变成独立的操作系统进程，某个 Tab 崩溃，只会影响该 Tab，其余一切正常。这是对 Web 应用时代基础设施的一次根本性修复。

**Omnibox**：Chrome 将此前浏览器上分离的 URL 栏和搜索栏合并为一个输入框。用户不再需要思考"我是要输入网址，还是要搜索"——只管输入，Chrome 帮你判断。这一设计与谷歌的商业模式高度契合：每一次未输入明确 URL 的操作，都默认触发谷歌搜索。

**沙盒安全**：每个 Tab 运行在独立的沙盒环境中，无法访问其他 Tab 或本地系统资源。这一设计大幅降低了浏览器恶意软件的感染面，彻底改变了"上网就是在拿个人电脑冒险"的时代。

**市场份额的逆转**

Chrome 发布时，Internet Explorer 占有约 70% 的浏览器市场份额，Firefox 占据了大部分剩余份额。谷歌将漫画册传递给技术博主和极客读者，让他们在感恩节回家时，帮父母把家里的电脑从 IE 换成 Chrome。

结果超出所有人的预期：

- Chrome 上线 18 个月后：4000 万用户
- 2010 年：7000 万用户，Chrome 和 IE 各占约 30% 市场份额，IE 从 70% 一路滑落
- 2012 年：2 亿用户，四年完成逆转
- 2014 年：Chrome 40%，IE 15%——战局已定
- 今日：Chrome 约 70% 市场份额，Safari（主要来自 iPhone）约 20%，IE 实质已死

Chrome 的开源版本 Chromium 免费提供给任何人，今天几乎所有非 Safari 浏览器都建立在 Chromium 之上。谷歌之所以愿意开源，原因很简单：不管谁来做浏览器，只要不是微软独家掌控，谷歌就能以"流量获取费用"的方式保住搜索入口。无论如何，这比让微软独吞浏览器市场要划算得多。

Sundar Pichai 领导了 Chrome 项目的全程。这个项目的成功，是他日后坐上谷歌 CEO 宝座最重要的敲门砖。

用一句话来总结：**Chrome 拯救了开放网络**。它是现代 Web 作为应用平台得以延续的关键原因。

## 第七章：Android——移动时代的生存之战

**安迪·鲁宾（Andy Rubin）的漫长旅程**

安迪·鲁宾（Andy Rubin）曾在苹果工作，随后参与创立了 General Magic——一家在 1990 年代初就试图做出智能手机的传奇失败创业公司，比 iPhone 早了整整 15 年。

General Magic 解体后，安迪创立了 Danger，并打造出 Sidekick——一款以消息功能为核心的智能手机，与 T-Mobile 合作上市，在名人圈中掀起一股风潮，成为 Blackberry 之外最重要的早期智能手机形态之一。值得一提的是，广岛锁三（Hiroshi Lockheimer）是 Danger 的第一位员工，他日后将在谷歌长期主导 Android 和 Chrome 两大平台。

2003 年，安迪创立了一家名为 Android 的公司，最初的目标是为数码相机开发一套跨平台的开源操作系统——因为他认为，数码相机将是下一个重要的消费电子品类，而那个时代的相机软件糟糕透顶。但很快，他意识到手机才是真正的未来：手机会把相机功能吃掉，而不是相反。Android 的目标随即从相机转向智能手机。

他开始向手机制造商和运营商兜售这套免费开源的移动操作系统，但收效甚微——一家只有八个人、估值约 $10 million 的小公司，没有任何运营商愿意认真对待。

**$50 million 买下移动时代的未来**

2005 年 7 月，资金即将告罄的安迪与拉里·佩奇见了面。拉里的反应出乎所有人意料："不用再融资了，我直接买你。"

谷歌以 $50 million 收购了 Android。

表面上看，这笔钱很少。但真正的投入是此后数十亿美元的研发费用。谷歌之所以动作如此迅速，是因为他们已经嗅到了危险：移动互联网正在加速到来，谷歌在这个平台上的处境，比在 PC 互联网上更加脆弱。如果手机平台落入微软或苹果之手，谷歌的搜索帝国将面临灭顶之灾。

彼时的智能手机市场，Blackberry 是主流，微软的 Windows Mobile 是面向手机制造商的授权操作系统，Nokia 是全栈整合的巨头。所有人都过得不错，没有人真正想打破现状——直到史蒂夫·乔布斯（Steve Jobs）和安迪·鲁宾同时决定改变这一切。

**iPhone 亮相，一切重来**

2007 年 1 月，苹果在乔布斯的历史性主题演讲中发布了 iPhone。埃里克·施密特甚至被邀请上台，开玩笑说两家公司合并的话可以叫"Apple Goo"。台下的乔布斯强撑着笑脸，眼神中已经藏不住不悦。

就在那天，Android 内部的"Sooner"原型机——一款类 Blackberry 的实体键盘手机——被直接扔进了垃圾桶。触摸屏才是未来，"Dream"原型机从此成为 A 队。

乔布斯的愤怒来得很快。他在苹果内部会议上公开宣称："我们没有进入搜索领域，但他们进入了手机领域。谷歌想杀死 iPhone，我们不会允许。"在沃尔特·艾萨克森（Walter Isaacson）的传记 *Steve Jobs* 中，乔布斯说："我愿意动用苹果银行账户上所有的 $40 billion，我愿意花光最后一口气，为此打一场热核战争（thermonuclear war）。"

两家公司随即陷入专利战，焦点在于多点触控手势（multitouch）。因此，早期的 Android 手机在好几年内都没有捏合缩放（pinch to zoom）功能，并在屏幕底部设置了四个实体导航按键，以绕开苹果的手势专利。

然而，Android 并非一无是处。它率先推出了应用市场（即后来的 Play Store），推出了下拉通知栏，支持多任务处理，允许用户拖拽调整主屏图标排列——这些功能，苹果都花了数年才补上。

**摩托罗拉 Droid：真正赢得战争的那一刻**

2007 年 11 月，谷歌宣布成立开放手机联盟（Open Handset Alliance），汇聚了 HTC、摩托罗拉、三星、LG、T-Mobile、Sprint、高通、英特尔等数十家公司。

2008 年 9 月，第一款 Android 手机 T-Mobile G1（制造商为 HTC，内部代号 Dream）正式上市。发布现场颇为反传统：拉里和谢尔盖穿着休闲装，脚踩溜冰鞋滑进了活动场地。G1 最终销量约 100 万部，而 iPhone 在 2008 年一年就卖出了 1100 万部。

真正的战局逆转，发生在 2009 年圣诞季——摩托罗拉 Droid 上市。

Verizon 当时正被 AT&T 的 iPhone 打得焦头烂额，最优质的用户纷纷流向对手。Verizon 决定孤注一掷，全面押注 Android：选中摩托罗拉 Droid 作为旗舰机，投入巨额营销预算，并授权使用《星球大战》中的机器人音效。

广告创意极具攻击性：前 80% 模仿苹果广告的风格，用优雅的白底字幕列出 iPhone 的缺点——"我不支持多任务"、"我的电池不可更换"……最后五秒突然黑屏，字幕变成冷峻的 "Droid does"（Droid 做到了）。

Droid 真正致命的卖点，是一个软件功能：**Google Maps 逐步导航**。在 Droid 之前，专用 GPS 导航设备（Garmin、TomTom）还是一个繁荣的消费电子品类，用户每月还要额外支付订阅费。Droid 上线那天，这整个品类实质上被消灭了——Google Maps 的逐步语音导航比任何独立 GPS 设备都更准确、更实时、完全免费，还随时更新。

2009 年圣诞季，Android 全球市场份额约为 5-6%。一年后，这个数字变成了 30%。2011 年达到 50%，2013 年突破 80%。

**"低于零成本"商业模式**

分析师比尔·格利（Bill Gurley）在一篇博客中将谷歌的 Android 策略命名为"低于零成本商业模式"（The Less Than Free Business Model）：

Android 本身是免费的、开源的，任何人都可以拿去用。但谷歌还走了更进一步——它向手机制造商和运营商支付流量获取费用（traffic acquisition costs），换取谷歌搜索在设备上的默认地位。

如果手机制造商想要接入谷歌的整套生态——Play Store 应用市场、Gmail、Google Maps、YouTube——就必须将谷歌搜索设为默认。这就是那个"无法拒绝的提案"：你可以拿走免费的操作系统，但如果你想让用户真正满意，就得把搜索入口给我。

这套逻辑对微软造成了毁灭性打击：微软的策略是向制造商收取许可费，而谷歌不仅免费给，还倒贴钱——没有任何制造商会选择前者。Palm 和 Blackberry 也没能幸存。

今日，全球超过 30 亿台活跃 Android 设备运行在这套体系下。谷歌每年向 Android 生态的运营商和制造商支付约 $10 billion 的流量获取费用，换来的是对非苹果手机市场的全面覆盖，以及对平台命运的最终掌控权。

**科技史上的罕见成就**

比较科技史上几次伟大的平台更迭：IBM 统治大型机，在 PC 时代失势；微软统治 PC，在 Web 时代失去主导权；谷歌统治 Web，并**成功延伸进了移动时代**——这在科技史上几乎没有先例。

谷歌并不是靠卖手机赚钱，也不是靠卖 Android 系统赚钱，它赚的仍然是搜索广告。但通过 Android，它将这个商业模式无缝延伸到了下一个十亿级计算平台。

这就是 Android 真正的价值所在：不是 Play Store 的几十亿美元收入，而是在移动时代守住了搜索广告这台每年价值数千亿美元的印钞机。

## 第八章：Google Plus、Alphabet 与 AI 时代的黎明

**被遗忘的先行者：Orkut**

谷歌并非没有提前感受到社交网络的到来。

2004 年 1 月，就在 Gmail 发布前几个月，一位名叫 Orkut Büyükkökten 的土耳其裔工程师用 20% 自由时间打造了一个社交网络，以自己的名字命名——Orkut。这个社交网络在美国毫无存在感，却意外在巴西和印度爆发，峰值时期积累了约 3 亿用户。谷歌对此的态度是：还不错，但不重要。

2007 年，谷歌推出了 Open Social，试图构建一个开放的社交协议——但 Facebook 拒绝参与，而 Facebook 就是社交本身，其余的加起来也无济于事。2009 年，Google Wave 以一段令人叹为观止的 demo 引发全球技术圈的热议，却最终因为没人知道该拿它做什么而悄然关闭。2010 年，Google Buzz 因严重的隐私问题在上线后没多久便仓皇下架。

**Google Plus：宫廷政治与战略焦虑**

2010 年，谷歌高层人员被集中召集，讨论应对社交媒体的战略。随后，时任工程高级副总裁乌尔斯·霍尔茨勒（Urs Hölzle）写下了一份备忘录，警告称互联网正在从"页面与应用"的世界，转变为"以人为核心"的世界——这对谷歌是一个潜在的存在性威胁。

但这一场战役的真正动因，或许并非来自对 Facebook 的战略恐惧，而更多是一场宫廷政治。

彼时的谷歌，已经不再是那个铁板一块的小公司。Android 是一块独立的飞地，Chrome 是另一块，Search 有自己的护城河，YouTube 完全自成一体，Gmail 是唯一拥有用户身份认证的产品。公司内部充斥着争权夺利的大佬们，各自坐镇一方诸侯，公司整体方向开始模糊。

2011 年 4 月，拉里·佩奇重新出任谷歌 CEO，将办公室搬进了即将成为 Google Plus 大本营的楼栋。Google Plus，更像是他用来重新统一公司的一面旗帜——无论最终选择了哪个产品来完成这件事，代价都是巨大的。而不幸的是，他们选了这个。

**文化毒药**

Google Plus 由维克·冈多特拉（Vic Gundotra）主导，在一年内以命令式自上而下的方式强行推进，与谷歌此前"工程师激情驱动"的产品文化完全背道而驰。

公司内所有团队的 OKR（目标与关键结果）都与 Google Plus 的推广指标挂钩。数十个团队被强制从各自的产品中抽调人力支援 Plus。YouTube 的评论区被强制整合进 Google Plus 发布系统——用时任谷歌实习生（后来的 TechCrunch 主编）丹尼·克里希顿（Danny Crichton）的话说，整个公司的文化在那段时间变得"深度有毒"。谷歌甚至在移动广告展示单元中加入了"Plus 一下"按钮——没有任何用户会去点赞一个广告。

**产品的失败，以及更深的战略失误**

作为产品，Google Plus 并非一无是处：它提出了"圈子（Circles）"功能来精细管理好友分组，推出了 Hangouts 视频通话，孵化了后来成为独立产品的 Google Photos。但从用户角度来说，没有人需要一个有 Facebook 的情况下还要再多一个 Facebook。

更糟糕的是，2011 年 Google Plus 上线时，马克·扎克伯格（Mark Zuckerberg）已经开始将 Facebook 转型：买下 Instagram，买下 WhatsApp，将整个社交网络从"与认识的人分享"的模式，重新定义为"私信 + 公共媒体"的双轨结构。谷歌追的，是一个正在消亡的范式。

Google Plus 的失误，在事后看来，让谷歌错过了两件真正重要的事：

**即时通讯**。在 Google Plus 如火如荼的那几年，Facebook 以 $19 billion 收购了 WhatsApp。埃里克·施密特当时恰好在斯坦福教课，收购消息传来时，他进入教室的第一句话是："我们彻底错过了。"

**云计算**。谷歌彼时应当将全部组织精力投入到企业级云服务的建设上，却将最宝贵的工程和管理资源耗在了一个注定失败的社交产品上。这正是为什么谷歌云（Google Cloud）至今仍落后于亚马逊 AWS 和微软 Azure 的根本原因之一。

2014 年，维克·冈多特拉离开谷歌。2019 年，Google Plus 以"发现重大安全漏洞"为由正式关闭，留下的是漫天散落在互联网角落里的死链。

**然而，失败也留下了遗产**

Google Plus 的失败有两个意外的正面结果。

第一，谷歌账号（Google Account）体系。今天，谷歌所有的产品、设备和服务，都通过同一套账号系统打通——这在 Google Plus 之前根本不存在。统一的身份认证，是这场代价惨烈的实验留下的最重要遗产之一。

第二，Google Photos 和 Google Meet（前身 Hangouts）——两个在 Google Plus 废墟上诞生的产品，前者如今拥有超过 10 亿用户，后者则在疫情期间成为了谷歌视频会议的核心基础设施。

而社交媒体本身的走向，最终对谷歌也并非如当初担忧的那样致命——整个社交媒体的内容消费模式，最终演化成了 YouTube 所代表的那种形态：算法驱动、以视频为核心的公共内容消费。谷歌什么都没有做，最后赢了。

**Alphabet：一场组织重生**

2015 年 8 月，谷歌宣布重组为 Alphabet 控股公司。

拉里·佩奇担任 Alphabet CEO，谢尔盖·布林（Sergey Brin）担任总裁，桑达尔·皮查伊（Sundar Pichai）出任谷歌 CEO——接管搜索、Gmail、YouTube、Maps、Android、Chrome 等全部核心业务。

所谓"其他投注"（Other Bets）业务，包括 Waymo（自动驾驶）、Nest（智能家居）、Google Fiber（宽带）、Calico（寿命研究）、Verily（生命科学）、Google X（前沿研究实验室）以及 Google Ventures 和 Capital G 两支投资基金，被独立拆分出来，各自运营。

这一重组有多重含义：拉里的退出，让谷歌核心业务从他的直接管理中解放出来；桑达尔的接任，是对 Chrome 和 Android 时代的奖赏，也是对公司文化的一次修复——他被广泛认为是一位善于调和大自我冲突的领导者。

2015 年，Alphabet 整体营收约 $75 billion，其中 $52 billion 来自谷歌核心产品，约 $15 billion 来自 DoubleClick/AdSense 网络。谷歌运营利润约 $23 billion，其他投注业务亏损约 $3.5 billion。核心商业逻辑，与 2002 年相比，没有实质性变化。

**写在 AI 时代黎明之前**

2015 年到 2016 年间，以下这些人同时是谷歌的员工：

亚历克斯·克里热夫斯基（Alex Krizhevsky），AlexNet 的发明者；杰弗里·辛顿（Geoffrey Hinton），"AI 教父"；伊利亚·苏茨克维尔（Ilya Sutskever），OpenAI 联合创始人；达里奥·阿莫代伊（Dario Amodei），Anthropic 联合创始人；安德烈·卡帕西（Andrej Karpathy），特斯拉 AI 首席科学家；伊恩·古德费洛（Ian Goodfellow），GAN 发明者；以及 DeepMind 的三位联合创始人——德米斯·哈萨比斯（Demis Hassabis）、谢恩·雷格（Shane Legg）和穆斯塔法·苏莱曼（Mustafa Suleyman）。

他们都曾是谷歌的员工。同一时期，谷歌的 Brain 团队还在默默撰写一篇日后将震动整个人工智能领域的论文，发表于 2017 年 6 月，题为 *Attention Is All You Need*——这篇论文提出了 Transformer 架构，而 ChatGPT 中的字母 T，正是 Transformer 的首字母。

这一切的聚合，并非偶然。谷歌过去二十年所做的一切——索引整个互联网，存储数十亿人的邮件、照片和视频，构建全球最大的分布式计算基础设施——在事后看来，都是为 AI 时代积累数据、算力和人才的漫长蓄力。

拉里·佩奇在 2000 年说过这样一段话：

> "人工智能将会是谷歌终极版本。如果我们拥有终极搜索引擎，它将理解网络上的一切，理解你真正想要什么，并给出正确答案。这显然就是人工智能……我们还远没有做到这一点，但我们可以一步步接近它。"

这是 25 年前的预言。

2015 年的 Alphabet，是这段漫长旅程的一个中间站，而不是终点。下一章，才刚刚开始。
